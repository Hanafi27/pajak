                                                        ABSTRAK  
 
 
Perkembangan teknologi informasi mendorong instansi pemerintah untuk meningkatkan 
kualitas pelayanan melalui penerapan sistem berbasis digital. Badan Pendapatan Daerah 
(BAPENDA) Kabupaten Bandung sebagai instansi yang mengelola Pajak Bumi dan 
Bangunan (PBB) memerlukan sistem yang mampu membantu proses pengolahan data 
secara efektif, akurat, dan terintegrasi. Proses pengelolaan data PBB yang masih 
dilakukan secara manual atau semi-komputerisasi berpotensi menimbulkan kesalahan 
pencatatan, duplikasi data, serta keterlambatan dalam penyajian laporan. Oleh karena itu, 
diperlukan suatu sistem informasi berbasis web yang dapat mendukung pengolahan data 
PBB secara terstruktur dan sistematis. 
Penelitian ini bertujuan untuk merancang sistem pengolahan data Pajak Bumi dan 
Bangunan berbasis web yang dapat membantu meningkatkan ketepatan data serta 
efisiensi kerja pada BAPENDA Kabupaten Bandung. Metode penelitian yang digunakan 
meliputi observasi, wawancara, dan studi pustaka untuk mengidentifikasi kebutuhan 
sistem. Perancangan sistem dilakukan menggunakan pendekatan Unified Modeling 
Language (UML) yang meliputi Use Case Diagram, Activity Diagram, Sequence 
Diagram, dan Class Diagram, serta perancangan basis data menggunakan Entity 
Relationship Diagram (ERD). 
Hasil dari penelitian ini berupa rancangan sistem berbasis web yang memiliki fitur login, 
pengelolaan data wajib pajak, pengelolaan data objek pajak, pengolahan data PBB, serta 
pembuatan laporan. Sistem dirancang menggunakan framework Laravel dan database 
MySQL untuk memastikan keamanan serta integritas data. Pengujian sistem dilakukan 
menggunakan metode Black Box Testing yang menunjukkan bahwa seluruh fitur sistem 
berjalan sesuai dengan kebutuhan fungsional yang telah ditentukan. 
Dengan adanya sistem ini, diharapkan proses pengolahan data Pajak Bumi dan Bangunan 
pada BAPENDA Kabupaten Bandung menjadi lebih cepat, akurat, dan efisien, serta 
mampu meminimalisir kesalahan dalam pencatatan dan pelaporan data pajak. 
Kata Kunci: Sistem Informasi, Pajak Bumi dan Bangunan, BAPENDA, Laravel, Basis 
Data

                                                          ABSTRACT 
                                                                     
The development of information technology encourages government institutions to 
improve service quality through the implementation of digital-based systems. The 
Regional Revenue Agency (BAPENDA) of Bandung Regency, as the institution 
responsible for managing Land and Building Tax (PBB), requires a system capable of 
supporting data processing effectively, accurately, and in an integrated manner. The 
existing data management process, which is still carried out manually or semi-
computerized, has the potential to cause recording errors, data duplication, and delays in 
report generation. Therefore, a web-based information system is needed to support 
structured and systematic PBB data processing. 
This study aims to design a web-based Land and Building Tax data processing system to 
improve data accuracy and work efficiency at BAPENDA Bandung Regency. The 
research methods used include observation, interviews, and literature study to identify 
system requirements. The system design was carried out using the Unified Modeling 
Language (UML) approach, including Use Case Diagrams, Activity Diagrams, Sequence 
Diagrams, and Class Diagrams, as well as database design using an Entity Relationship 
Diagram (ERD). 
The result of this research is a web-based system design equipped with features such as 
login authentication, taxpayer data management, tax object management, PBB data 
processing, and report generation. The system was designed using the Laravel framework 
and MySQL database to ensure data security and integrity. System testing was conducted 
using the Black Box Testing method, and the results indicate that all system features 
function properly according to the defined functional requirements. 
With the implementation of this system, it is expected that the Land and Building Tax 
data processing at BAPENDA Bandung Regency will become faster, more accurate, and 
more efficient, while minimizing errors in data recording and reporting. 
Keywords: Information System, Land and Building Tax, BAPENDA, Laravel, Database

Menurut Mulyani (2017), perancangan sistem adalah proses penentuan data, 
proses, dan tampilan yang diperlukan oleh sistem baru. Perancangan sistem memberikan 
gambaran secara rinci mengenai bagaimana sistem akan bekerja, mulai dari alur proses,

2.2 Sistem Informasi 
Sistem informasi merupakan sekumpulan komponen yang saling berhubungan dan 
bekerja sama untuk mengumpulkan, mengolah, menyimpan, serta menyajikan informasi 
yang dibutuhkan oleh pengguna. Sistem informasi berperan penting dalam mendukung 
kegiatan operasional dan pengambilan keputusan dalam suatu organisasi.  Menurut Jonny 
Seah (2020), sistem informasi adalah kombinasi dari perangkat keras, perangkat lunak, 
basis data, prosedur, dan sumber daya manusia yang saling berinteraksi untuk 
menghasilkan informasi. Sementara itu, Wahyudi dan Ridho (2020) menyatakan bahwa 
sistem informasi meru pakan sistem yang dirancang untuk menyediakan informasi guna 
mendukung aktivitas organisasi.  Dalam pengelolaan Pajak Bumi dan Bangunan, sistem 
informasi digunakan untuk membantu proses pencatatan dan pengolahan data pajak agar 
lebih tertata dan mudah diakses. Dengan adanya sistem informasi yang terintegrasi, potensi

2.3 Sistem Informasi Akuntansi 
Sistem Informasi Akuntansi (SIA) merupakan bagia n dari sistem informasi yang 
berfokus pada pengolahan data keuangan dan akuntansi. Sistem ini berfungsi untuk 
mengumpulkan, mencatat, mengolah, dan menyajikan informasi keuangan yang 
dibutuhkan oleh manajemen dan pihak terkait. Menurut Rahmawati dan Sumarno (2020), 
Sistem Informasi Akuntansi memiliki peran penting dalam menunjang kegiatan 
operasional, membantu pengambilan keputusan, serta meningkatkan pengendalian internal 
organisasi. Sementara itu, Wardani dan Dewi (2022) menyatakan bahwa SIA berperan 
dalam menghasilkan informasi akuntansi yang akurat, relevan, dan tepat waktu.  Dalam 
pengelolaan Pajak Bumi dan Bangunan pada BAPENDA Kabupaten Bandung, Sistem 
Informasi Akuntansi berperan dalam mendukung ketepatan data penerimaan pajak, 
membantu proses pencatatan transaksi pajak, serta menghasilkan laporan penerimaan pajak 
daerah yang dapat dipertanggungjawabkan. 
2.4 Pajak Bumi dan Bangunan 
Pajak Bumi dan Bangunan (PBB) merupakan pajak yang dikenakan atas

3.1 Analisis Sistem 
Analisis sistem merupakan tahap awal dalam pengembangan sistem yang bertujuan 
untuk memahami kondisi sistem yang sedang berjalan serta mengidentifikasi permasalahan 
yang ada. Tahap analisis ini dilakukan agar sistem yang dirancang dapat menjawab 
kebutuhan pengguna dan memperbaiki kelemahan pada sistem sebelumnya.  Pada kerja 
praktek ini, analisis  sistem difokuskan pada proses pengolahan data Pajak Bumi dan 
Bangunan (PBB) di BAPENDA Kabupaten Bandung. Analisis dilakukan melalui observasi 
langsung, wawancara dengan petugas terkait, serta pengamatan terhadap dokumen dan alur

3.1.1 Analisis Sistem yang Sedang Berjalan 
Berdasarkan hasil observasi yang dilakukan selama pelaksanaan kerja praktek di 
BAPENDA Kabupaten Bandung, diketahui bahwa proses pengolahan data Pajak Bumi dan 
Bangunan masih belum sepenuhnya terintegrasi dalam satu sistem informasi. Proses 
pengolahan data dilakukan melalui beberapa tahapan yang terpisah, mulai dari penerimaan 
data wajib pajak, pencatatan data objek pajak, hingga penyusunan laporan. 
Karakteristik sistem yang sedang berjalan antara lain: 
1. Data Pajak Bumi dan Bangunan belum dikelola dalam satu sistem yang terintegrasi. 
2. Proses pencatatan dan pengolahan data dilakukan secara bertahap sehingga 
membutuhkan waktu yang relatif lama. 
3. Potensi terjadinya ketidaksesuaian data masi h cukup tinggi, terutama pada saat 
proses rekapitulasi data. 
4. Penyajian laporan data PBB belum sepenuhnya mendukung kebutuhan informasi

5. Belum tersedia sistem berbasis web yang dapat diakses secara fleksibel oleh 
petugas terkait.

sebuah sistem aplikasi yang dapat digunakan untuk mengelola data Pajak Bumi dan 
Bangunan (PBB). Sistem ini dirancang untuk membantu proses pencatatan, pengolahan, 
dan pelaporan data PBB agar lebih terstruktur, terintegrasi, dan meminimalisir kesalahan

masing. 
4.2 Spesifikasi Implementasi Sistem 
          Dalam pengembangan sistem ini, dibutuhkan perangkat keras dan perangkat lunak 
untuk mendukung jalannya aplikasi.  Perangkat keras diguna kan sebagai media untuk 
menjalankan sistem, sedangkan perangkat lunak berfungsi sebagai platform dalam 
pengembangan dan pengoperasian aplikasi. Spesifikasi yang digunakan harus mampu 
mendukung kinerja sistem agar dapat berjalan dengan baik dan optimal. Sel ain itu, 
pemilihan perangkat yang sesuai juga bertujuan untuk memastikan proses pengolahan data 
dapat dilakukan secara cepat dan efisien. Dengan dukungan perangkat yang memadai, 
sistem diharapkan dapat berjalan secara stabil dan memenuhi kebutuhan pengguna.

akses hanya kepada pihak yang berwenang. 
Adapun implementasi antarmuka sistem meliputi: 
1.Halaman Login  Halaman login digunakan untuk proses autentikasi pengguna. Pengguna

langsung mengakses menu yang dibutuhkan.

5.1 Kesimpulan  
Berdasarkan hasil analisis, perancangan, implementasi, dan pengujian sistem 
pengolahan data Pajak Bumi dan Bangunan berbasis web, maka dapat disimpulkan 
bahwa: 
1. Sistem yang dibangun mampu membantu proses pengelolaan data PBB secara 
lebih terstruktur dan terkomputerisasi. 
2. Proses penginputan, pencarian, dan pengolahan data menjadi lebih cepat dan 
efisien dibandingkan dengan sistem manual. 
3. Sistem mampu meminimalkan kesalahan pencatatan serta meningkatkan akurasi 
data. 
4. Pembuatan laporan menjadi lebih mudah dan dapat dilakukan dalam waktu yang 
lebih singkat. 
 
5.2  Saran  
Adapun saran untuk pengembangan sistem selanjutnya adalah sebagai berikut: 
1. Sistem dapat dikembangkan dengan fitur pembayaran pajak secara online agar 
lebih modern dan terintegrasi. 
2. Perlu ditambahkan sistem backup data otomatis untuk menjaga keamanan dan 
keutuhan data. 
3. Sistem dapat dikembangkan dengan fitur notifikasi jatuh tempo pembayaran 
pajak. 
4. Diperlukan pelatihan kepada pengguna agar sistem dapat digunakan secara 
maksimal dan optimal.
